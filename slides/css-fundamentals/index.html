What is CSS?

Fully known as Cascading Style Sheets

Describes how a page should be styled for a particular document

Helps seperate style and presentation from content

You can include styles in several different ways...

Inline

bacon

Embedded

External

<link rel="stylesheet" href="style.css" />

However, please avoid using inline and embeded styles when possible

This...

<link rel="stylesheet" href="style.css" />

...is what you want to use to include stylesheets.

Embeded and inline styles can get messy and difficult to maintain. Additionally, by linking stylesheets we're seperating content and style

Cascading

Styles can cascade through different stylesheets and take priority based on the following order:

  1. Browser default styles Every browser has a default stylesheet
  2. User-specified stylesheets Some browsers allow users to create their own stylesheets
  3. External stylesheet Linked or referenced stylesheets
  4. Internal stylesheet Style tags listed in the document
  5. Inline styles Styles listed on an element

If any of the styles has the same selectors or priority, the latest one will "win"

This is what CSS looks like...

em {
  font-size: 12px;
}

Rule

In CSS, this is a rule. It contains everything telling the browser what elements to apply the given styles to

Selector

The selector is an expression that matches elements on the page to apply styles contained within the following block. This can be a number of different things, but we'll go over that in a minute

Declaration Block

Styles are grouped in blocks, starting and ending with opening and closing braces.

Property

The property part of the rule specifies what style property to apply

Value

The value part of the rule specifies the property value. The valid values you can put here all depend on the type of property you are using

Inheritance

Child elements inherit styles from parents and ancestors

p {
  color: red;
}


span {
  font-weight: bold;
}

<p>
  <span>Bacon</span> ipsum dolor sit amet brisket
  <span>pork</span> belly
  <span>shank</span> tri-tip.
</p>

Basic Selectors

Element Selector

Matches any element with the correspoding element type

</div>

More on Attribute Selectors

You can be more specific about the types of values you want to match

  • [attr] Element has attribute
  • [attr=value] Attribute must match value
  • [attr*=value] Attribute contains the value somewhere
  • [attr^=value] Attribute starts with the value
  • [attr$=value] Attribute ends with the value
  • [attr~=value] Attribute is in a space seperated list

Font Families

Specify a prioritized list of fonts for the matching elements

Browsers will select the first font, from left to right, that is available. Fonts with spaces must be "quoted"

<p><span class="serif">Bacon ipsum</span> dolor sit amet <span class="sans-serif">ham hock</span> jerky shank</p>

Generic Fonts

Always include at least one generic font in your font-family as a fallback

serif
A typeface with finished strokes, flared, tapering ends, or actual serifed endings
sans-serif
A typeface with plain stroke endings
monospace
A fixed-width typeface, typically used for code
cursive
A typeface with joining strokes or other cursive characteristics
fantasy
A primary decorative typeface
